# Contributing to Dotnet AI Engineer Roadmap

Thank you for your interest in contributing to this project! This repository is a comprehensive learning resource for AI Engineering using the .NET ecosystem, and contributions are welcome.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest improvements.
- Clearly describe the issue, including steps to reproduce if applicable.
- Reference the specific Week/Day/Topic where the issue exists.

### Suggesting Enhancements
- Open a GitHub Issue with the `enhancement` label.
- Describe the improvement and why it would benefit learners.

### Submitting Changes
1. **Fork** the repository.
2. **Create a branch** from `main` for your changes:
   ```bash
   git checkout -b feature/week-03-day-02-improvement
   ```
3. **Make your changes** following the conventions below.
4. **Commit** with a clear message:
   ```bash
   git commit -m "docs(week-03): add advanced chunking strategies example"
   ```
5. **Push** to your fork and open a **Pull Request**.

## Conventions

### Folder Structure
- Each day follows the pattern: `Week-XX-Name/Day-XX-Topic/`
- Documentation goes in `README.md` and supporting `.md` files.
- Code samples go in a `src/` subdirectory with a proper .NET project.

### Documentation Style
- Use clear, beginner-friendly language.
- Include code examples with syntax highlighting.
- Add references and further reading links.
- Use Mermaid diagrams for architecture explanations.

### Code Style
- Follow standard C# coding conventions.
- Use .NET 8+ features.
- Include XML documentation comments on public APIs.
- Keep examples focused and self-contained.

### Commit Messages
Follow [Conventional Commits](https://www.conventionalcommits.org/):
- `docs(week-01): add embedding theory explanation`
- `feat(week-03): add PDF chunking utility`
- `fix(week-04): correct cosine similarity formula`

## Code of Conduct

Be respectful, inclusive, and constructive. We're all here to learn.

## Questions?

Open a GitHub Discussion or Issue — happy to help!
